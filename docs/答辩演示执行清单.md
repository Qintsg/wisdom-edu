# 答辩演示执行清单

## 1. 启动顺序

### 1.1 后端

```bash
cd backend
.venv\Scripts\python.exe manage.py runserver 127.0.0.1:8000
```

### 1.2 前端

```bash
cd frontend
npm run dev
```

## 2. 数据重建

```bash
cd backend
.venv\Scripts\python.exe tools.py rebuild-demo-data --course-name "大数据技术与应用"
```

执行后至少确认以下事实：

- `teacher` 可登录教师端
- `student2` 为“已入班、零轨迹”的新生账号
- `student` 已具备固定评测报告与学习路径
- Neo4j 图谱同步成功

## 3. 演示课程导入包生成

```bash
cd backend
.venv\Scripts\python.exe tools.py generate-demo-course-archive --output "../output/答辩演示课程导入包.zip"
```

建议把 ZIP 文件提前放到桌面，现场选择文件更快。

## 4. 标准浏览器演练命令

### 4.1 预检链路

```bash
cd backend
.venv\Scripts\python.exe tools.py browser-audit --scenario prepare-defense-demo --frontend-url http://127.0.0.1:3000 --api-base-url http://127.0.0.1:8000
```

### 4.2 模拟正式答辩

```bash
cd backend
.venv\Scripts\python.exe tools.py browser-audit --scenario simulate-defense-demo --frontend-url http://127.0.0.1:3000 --api-base-url http://127.0.0.1:8000
```

> 当前标准答辩链路固定为：`teacher -> student2 -> student`。

## 7. 标准点击顺序

1. `teacher` 登录
2. 进入 `课程管理`
3. 点击 `导入建课`
4. 上传 `答辩演示课程导入包.zip`
5. 创建课程并进入工作区
6. 打开题库管理并编辑一道题
7. 退出登录
8. `student2` 登录
9. 展示学生仪表盘的“零轨迹”状态
10. 点击 `前往初始评测`
11. 展示三项测评均未完成
12. 退出登录
13. `student` 登录
14. 进入 `初始评测`
15. 点击 `查看评测报告`
16. 从报告进入 `学习路径`
17. 展示前三个学习节点
18. 进入 `阶段测试：大数据基础综合`
19. 按预定答案提交
20. 返回学习路径，展示刷新后的后续节点
21. 依次展示 `AI助手 / 在线作业 / 知识图谱 / 课程资源`

- 第 2 题：`B`

## 9. 现场注意事项

- 正式讲解不要使用 `student1`，避免和当前标准答辩口径冲突。
- `student2` 只展示新生入口，不在现场完整做完三套初始测评。
- `student` 用于固定讲解报告、路径、阶段测试和扩展页面。
- 如果导入建课等待稍长，可先口头说明 ZIP 导入能力，再展示已导入课程的工作区。
- 如果阶段测试提交后路径刷新稍慢，先讲反馈报告，再切回路径等待刷新完成。
