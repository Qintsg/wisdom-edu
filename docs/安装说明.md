# 安装说明

> 最后更新：2026-04-28

本说明聚焦**本地开发环境**安装与验证。生产环境请优先阅读 `服务器部署说明.md`。

## 环境要求

- Python `3.12`
- Node.js `20+`
- PostgreSQL
- Neo4j（启用完整知识图谱 / GraphRAG 时推荐）

## 0. 一键安装（推荐）

仓库根目录提供：

- `install.py`
- `install.bat`

推荐直接在项目根目录执行：

```bash
python install.py
```

Windows 也可以执行：

```bash
install.bat
```

脚本会交互式收集并写入以下配置：

- Django `SECRET_KEY`
- PostgreSQL 连接信息
- Neo4j 连接信息
- GraphRAG 向量器与 Qdrant 路径
- `LLM_PROVIDER` / `LLM_MODEL` / `LLM_API_FORMAT`
- 当前默认提供方对应的 API Key
- 自定义兼容网关的 `BASE_URL + API_KEY`（当 `LLM_PROVIDER=custom` 时）
- `KT_DKT_META_PATH` / `COURSE_RESOURCES_DIR`

并按选项执行：

- 创建 `backend/.venv`
- 安装后端依赖
- 安装前端依赖
- 执行 `migrate`
- 执行 `collectstatic`
- 安装 Playwright Chromium
- 构建前端生产包

默认生成文件：

- `backend/.env`

说明：

- `install.py` 负责写入环境级配置与敏感信息；
- `backend/config.ini` 负责版本内默认值（例如默认模型、超时、GraphRAG 向量参数）；
- 运行时会先读 `.env`，再回退到 `config.ini`。

## 1. 手动安装后端

```bash
cd backend
py -3.12 -m venv .venv
.venv\Scripts\activate
pip install --upgrade pip
pip install -r requirements.txt
```

运行配置统一维护在 `backend/.env`。可以先基于模板整理一份：

- `backend/.env.example`

如需调整默认提供方、模型、超时与 GraphRAG 嵌入参数，也可以同步查看：

- `backend/config.ini`

开发启动：

```bash
.venv\Scripts\python.exe manage.py migrate
.venv\Scripts\python.exe manage.py runserver 127.0.0.1:8000
```

## 2. 手动安装前端

```bash
cd frontend
npm install
npm run dev
```

默认开发地址：

- 前端：`http://127.0.0.1:3000`
- 后端：`http://127.0.0.1:8000`

说明：

- 本地开发时，`frontend/vite.config.ts` 已内置 `/api`、`/media`、`/static`、`/ws` 代理到 `127.0.0.1:8000`，因此 `npm run dev` 无需额外配置 Nginx 或其他反向代理。
- 如需联调到其他后端地址，可直接设置 `VITE_DEV_BACKEND_ORIGIN`，或调整 `frontend/vite.config.ts` 中对应代理的 `target`；如需覆盖开发端口，可设置 `VITE_DEV_PORT`。
- 当前生产构建默认使用站点同源 `/api`、`/media`、`/static`、`/ws`，由线上 Nginx / OpenResty 反向代理到后端。
- 如果生产环境确需跨域直连其他后端地址，可通过 `VITE_BACKEND_ORIGIN` 覆盖后重新执行 `npm run build`。

## 6. 常见问题

### 6.1 后端启动失败

- 优先检查 `backend/.env` 是否存在且内容完整
- 确认使用的是 `backend/.venv`
- 确认 PostgreSQL 与 Neo4j 已启动
- 执行 `tools.py diagnose`

### 6.2 前端接口 404 / ECONNREFUSED

- 确认后端运行在 `127.0.0.1:8000`
- 确认 `frontend/vite.config.ts` 中的开发代理未被本地修改破坏
- 如需联调到其他后端地址，检查 `frontend/vite.config.ts` 中 `/api`、`/media`、`/ws` 的代理 `target`
- 如生产构建访问错了后端地址，检查 `frontend/src/api/backend.ts` 是否仍指向正确的后端入口

### 6.3 AI 助手或 WebSocket 不可用

- 确认后端已正常启动 ASGI 应用（本地 `manage.py runserver` 在当前项目中可承载该链路）
- 确认浏览器控制台没有 `/ws/student/ai/chat` 连接失败
- 确认 `backend/.env` 中的 `LLM_PROVIDER / LLM_MODEL / API Key / Base URL` 配置与实际提供方一致
- 如走反向代理，请改看 `服务器部署说明.md`

### 6.4 浏览器场景失败

- 先确保已执行 `npm install`
- 如首次运行 Playwright 缺浏览器，可执行：

```bash
cd frontend
npx playwright install chromium
```

- 如演示数据异常，按顺序重跑：
  1. `tools.py rebuild-demo-data`
  2. `tools.py browser-audit --scenario prepare-demo --headed`
  3. `tools.py browser-audit --scenario simulate-demo --headed`
