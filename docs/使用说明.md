# 使用说明

> 面向学生、教师、管理员，以及负责演示、联调和验收的开发人员。

## 访问入口

- 前端开发地址：`http://127.0.0.1:3000`
- 后端开发地址：`http://127.0.0.1:8000`
- Swagger：`http://127.0.0.1:8000/api/docs/`
- ReDoc：`http://127.0.0.1:8000/api/redoc/`
- OpenAPI Schema：`http://127.0.0.1:8000/api/schema/`
- 认证入口：`/login`、`/register`

## 学生端

学生端路由统一挂在 `/student` 下，核心页面来自 `frontend/src/router/routes/student.ts`。

### 教师端主要页面

| 路由 | 作用 |
| --- | --- |
| `/student/dashboard` | 学习中心首页 |
| `/student/course-select` | 课程选择 |
| `/student/assessment` | 初始测评入口 |
| `/student/assessment/knowledge` | 知识测评 |
| `/student/assessment/ability` | 能力评测 |
| `/student/assessment/habit` | 习惯问卷 |
| `/student/assessment/report` | 测评报告 |
| `/student/profile` | 学习画像 |
| `/student/resources` | 课程资源 |
| `/student/knowledge-map` | 知识图谱 |
| `/student/learning-path` | 学习路径 |
| `/student/task/:nodeId` | 任务学习 |
| `/student/ai-assistant` | AI 助手 |
| `/student/exams` | 在线作业 / 阶段测试 |
| `/student/exam/:examId` | 作业作答页 |
| `/student/feedback/:reportId` | 反馈报告 |
| `/student/classes` | 我的班级 |
| `/student/settings` | 个人设置 |

### 典型学习流程

1. 进入 `/student/course-select` 选择当前课程。
2. 在 `/student/assessment` 完成知识、能力与习惯三类初始测评。
3. 打开 `/student/profile` 查看学习画像与薄弱点。
4. 打开 `/student/knowledge-map` 查看知识图谱、知识点详情和图谱增强解释。
5. 在 `/student/learning-path` 生成个性化学习路径。
6. 逐个进入 `/student/task/:nodeId` 完成任务学习与资源消化。
7. 在 `/student/ai-assistant` 发起课程问答、GraphRAG 检索或图谱增强问答。
8. 在 `/student/exams` 完成阶段测试与在线作业，并到 `/student/feedback/:reportId` 查看反馈报告。

## 教师端

教师端路由统一挂在 `/teacher` 下，核心页面来自 `frontend/src/router/routes/teacher.ts`。

### 管理员端主要页面

| 路由 | 作用 |
| --- | --- |
| `/teacher/dashboard` | 教学中心 |
| `/teacher/courses` | 课程管理 |
| `/teacher/courses/create` | 创建课程 |
| `/teacher/courses/:courseId` | 课程详情 |
| `/teacher/courses/:courseId/workspace/knowledge` | 课程图谱工作区 |
| `/teacher/courses/:courseId/workspace/questions` | 课程题库工作区 |
| `/teacher/courses/:courseId/workspace/resources` | 课程资源工作区 |
| `/teacher/courses/:courseId/workspace/exams` | 课程作业工作区 |
| `/teacher/resources` | 资源库管理 |
| `/teacher/knowledge` | 知识图谱管理 |
| `/teacher/questions` | 题库管理 |
| `/teacher/exams` | 作业管理 |
| `/teacher/students/:studentId/profile` | 学生画像 |

### 典型教学流程

1. 在 `/teacher/courses` 创建课程或进入已有课程。
2. 进入课程工作区维护知识点、关系、课程资源与题库。
3. 根据课程内容配置作业、阶段测试和评分策略。
4. 查看学生画像、学习路径与阶段反馈，评估学习进展。
5. 如需演示或批量导入资源，可配合 `backend/tools.py` 的课程初始化与导入命令使用。

| 路由 | 作用 |

## AI 与知识追踪常用接口

后端 AI 入口主要定义在 `backend/ai_services/urls.py`。

### 学生端 AI 能力

- `/api/student/ai/profile-analysis`：学习画像分析
- `/api/student/ai/path-planning`：学习路径规划
- `/api/student/ai/resource-reason`：资源推荐理由
- `/api/student/ai/feedback-report`：反馈报告生成
- `/api/student/ai/chat`：学生 AI 对话
- `/api/student/ai/graph-rag/search`：GraphRAG 搜索
- `/api/student/ai/graph-rag/ask`：GraphRAG 问答

### KT 能力

- `/api/ai/kt/predict`：单用户掌握度预测
- `/api/ai/kt/model-info`：当前可用 KT 模型与运行时信息
- `/api/ai/kt/batch-predict`：批量掌握度预测
- `/api/ai/kt/recommendations`：基于掌握度的学习建议

## 开发、演示与验收常用命令

### 本地启动

```bash
cd backend
uv sync
uv run python manage.py runserver 127.0.0.1:8000

cd frontend
npm run dev
```

### 回归与体检

```bash
cd backend
uv run python manage.py check
uv run python tools.py api-regression --all --json

cd frontend
npm run typecheck
npm run build
```

### GraphRAG / KT / 演示数据

```bash
cd backend
uv run python tools.py build-rag-index
uv run python tools.py mefkt-status
uv run python tools.py rebuild-demo-data --course-name "大数据技术与应用"
uv run python tools.py browser-audit --scenario audit --frontend-url http://127.0.0.1:3000 --api-base-url http://127.0.0.1:8000
```
