# frontend/AGENTS.md

## 1. 适用范围

- 本文件适用于 `frontend/` 目录；工作时必须同时遵守根目录 `AGENTS.md`。
- 权限只看根目录中的 `AI_PERMISSION_LEVEL: DANGEROUS`。

---

## 2. 前端专项规则

- 技术栈基线为 `Vue3 + Fluent 2`；若仓库没有更强约定，优先保持 `Composition API`、强类型、清晰分层与强语义命名。
- 前端可见 UI 必须统一使用 **Fluent 2** 风格，不自行发明另一套视觉体系。
- 页面不直接承担复杂业务编排、复杂 RAG 调度、KT 核心计算、图数据库直连或 Prompt / Agent 决策核心逻辑。
- 推荐结构优先保持：`app/`、`router/`、`views/`、`components/`、`stores/`、`services/apis`、`models/types`、`utils/`。
- 涉及学习建议、推荐、掌握度解释、图谱路径、Agent 输出时，前端应尽量展示来源、依据、关键状态与限制说明，避免把模型输出包装成绝对结论。

---

## 3. 前端代码规范

- 优先遵循 Vue3、TypeScript 与仓库既有约定。
- 禁止 `var`、`any` 与无语义命名（如 `tmp`、`data`、`obj`、`res`）；组件、store、API 与 service 命名必须体现职责。
- 共享视觉元素优先抽成设计系统组件；不要在页面层反复硬编码颜色、间距、圆角与状态色。
- 注释率遵循根目录要求；单文件组件职责清晰，复杂页面拆分为局部组件，状态与 API 映射逻辑不要散落在模板中。
- 不保留空组件、假返回值或未接线交互；如确需保留未完成项，只能使用 `TODO:` / `FIXME:` 并说明原因与影响。

---

## 4. 验证与结果说明

- 前端改动优先验证：页面可打开、接口调用正确、关键交互可用、路由可用、Fluent 2 风格一致、图谱/推荐展示正确、控制台无关键报错。
- 若影响页面行为、路由、接口依赖、配置或平台说明，需要同步更新相关 `docs/` 文档。
- 完成前端任务时，说明中必须指出：改了哪些页面 / 组件 / 状态逻辑，是否影响学习流程、推荐展示、图谱展示、问答展示，是否修改 API 依赖，做了哪些验证，当前块 commit 与剩余风险是什么。
